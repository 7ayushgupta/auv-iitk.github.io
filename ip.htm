Image Processing

The team uses OpenCV to processs the Bottom Camera and front camera feed to realize the direction and velocity with which Bot moves to get the associated task done.

The image processing is done on the Odroid processor and the nodes doing this are in constant sync with nodes which are responsible for actuating motion in the Bot.

OpenCV with ROS
ROS provides us a great foundation to interact with OpenCV which in its own right, the easiest and most documented library. In this way our Vision nodes efforlessly react with our Motion Control nodes.
Breaking Image in Components
We break down the input into H,S and V components and then threshold them separately, after that we combine them so that we get a superposition of all of them which makes it more resistant to noise,hence more robust.
Using Convex Hull
We use Convex Hull on contours providing much smoother edges,ideal for the boi detection task which complements our min-enclosing functions helping us detect boi even when it is not completely in our frame.
